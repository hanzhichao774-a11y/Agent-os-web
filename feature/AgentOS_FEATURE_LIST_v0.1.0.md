### F-001 · 全局两栏布局与 Sidebar

**优先级：** P0
**模块：** 全局

**功能描述**
所有页面采用两栏结构：左侧 Sidebar（240px，可折叠为 56px）+ 右侧主内容区。Agent 对话区嵌入主内容区右列，不再有全局常驻第三栏。

**Sidebar 结构（Member）**
```
[Logo]  公司名称       [←折叠]
─────────────────────────────
○ 首页
○ Skills
○ Tools
─────────────────────────────
∨ 新建任务             [+]       ← 独立任务区（WorkAgent 驱动，无需归属项目）
    Q4 财报分析思路梳理
    供应商评估框架设计
    数据清洗方法论对比
─────────────────────────────
我的项目              [+]
▼ 供应链金融风控      ●
    财报分析 Q4
    供应商评估
▼ 智能客服知识中台
    FAQ优化
─────────────────────────────
[头像]  samhar        [Member]
```

**Sidebar 结构（Admin 额外增加）**
```
─────────────────────────────
─ 组织管理 ──────────────────
  成员管理
  团队管理
  用量管理
  系统设置
─────────────────────────────
[头像]  admin         [Admin]
```

**权限**
- Member 不可见「组织管理」导航组
- Admin 可见所有导航项
- 账号类型标识不可点击，无切换入口

**验收标准**

### F-002 · 账号登录与类型识别

**优先级：** P0
**模块：** 全局

**功能描述**
用户通过企业邮箱登录，系统根据账号类型（Member / Admin）渲染不同的首页和权限体系。账号类型由 IT 管理员在管理后台配置，登录后固定，不可自行切换。

**权限**
- Member：登录后渲染 HubAgent 首页
- Admin：登录后渲染 OrgAgent 首页
- Member 访问 Admin 专属路由（`/settings/*`、`/org/*`）→ 302 重定向首页

**上下文**
- 登录后，系统初始化该用户的 HubAgent 实例（若首次登录）
- Admin 登录后，系统检查 OrgAgent 实例是否存在（全组织唯一）

**验收标准**
- [ ] 登录成功后根据账号类型跳转不同首页
- [ ] Member 访问 Admin 路由被重定向
- [ ] 首次登录时 HubAgent 完成初始化

### F-003 · 首页数据看板 — Member

**优先级：** P0
**模块：** 首页 `/dashboard`

**功能描述**
Member 首页左列的数据看板，展示个人维度的跨项目数据概览。右列为 HubAgent 对话区（见 F-004）。

**布局**
```
┌──────────────────────┬──────────────────────┐
│  数据看板（左 60%）   │  HubAgent 对话区（右 40%）│
└──────────────────────┴──────────────────────┘
```

**KPI 卡片（4 张）**

| 卡片 | 数据范围 | 点击行为 |
|------|---------|---------|
| 本月 Token 消耗 | 该 Member 参与的所有项目 | 展开消耗明细 |
| 活跃项目 | 有进行中任务的项目数 | 跳转项目列表 |
| 任务成功率 | 该 Member 被派发并完成的任务（近 7 日）| 展开任务统计 |
| 待处理审批 | 该 Member 作为负责人的项目中待审批数 | 跳转 HubAgent 对话区 |

「待处理审批」仅在该 Member 是某项目负责人且有待审批时显示 `●`，普通成员此卡片显示空态。

**中部：项目消耗概览表**
- 列：项目名 / 本月消耗 / 占比进度条 / 趋势
- 仅显示该 Member 参与的项目（被派发过任务的项目）
- 按消耗量降序，点击行跳转项目页

**底部：近 7 日消耗折线图**

**上下文范围**
- 数据严格限定在该 Member 参与的范围内
- 不显示未参与项目的任何数据

**权限**
- 仅 Member 账号渲染此看板
- Admin 账号渲染 F-008 的全组织看板

**验收标准**
- [ ] KPI 卡片数据仅反映当前 Member 的个人维度
- [ ] 待处理审批角标与实际数同步，处理后消失
- [ ] 项目消耗表仅显示该 Member 参与的项目
- [ ] 数据加载中有 skeleton 占位

### F-004 · HubAgent 对话区

**优先级：** P0
**模块：** HubAgent（首页 / Skills / Tools 右列）

**功能描述**
HubAgent 对话区是 Member 与 HubAgent 交互的统一界面，在首页、Skills 页、Tools 页的右列复用同一组件。页面切换时对话历史保留，上下文随场景自动切换。

**上下文切换规则**

| 所在页面 | HubAgent 上下文模式 | 能力范围 |
|---------|-----------------|---------|
| 首页 | 全局模式 | 跨项目问答、审批处理、项目创建 |
| Skills 页 | Skill 管理模式 | Skill 创建引导、使用统计查询 |
| Tools 页 | Tools 管理模式 | Tool 连接配置、权限设置、费用查询 |

页面切换时，HubAgent 顶部显示短暂的上下文切换提示（「已切换到 Skill 管理模式」），2 秒后消失。

**记忆归属**
- 对话历史存入该 Member 的 HubAgent 记忆（个人私有）
- HubAgent 记忆不跨 Member 共享
- HubAgent 不可访问其他 Member 的记忆，不可访问 WorkAgent 的项目记忆

**权限**
- 仅 Member 账号有 HubAgent
- Admin 账号的首页为 OrgAgent，无 HubAgent

**界面结构**
```
┌──────────────────────────────────────┐
│ HubAgent                    ● 在线   │
├──────────────────────────────────────┤
│  对话区（流式输出，消息历史）          │
│  空状态：展示快捷引导问题             │
├──────────────────────────────────────┤
│ 输入消息...                     [→]  │
└──────────────────────────────────────┘
```

**验收标准**
- [ ] 流式输出正常渲染，支持 Markdown
- [ ] 页面切换时对话历史不丢失
- [ ] 上下文切换提示短暂显示后消失
- [ ] `Enter` 发送，`Shift+Enter` 换行

### F-005 · 新建任务页（独立任务，无需项目）

**优先级：** P1
**模块：** WorkAgent / 独立任务

**功能描述**
Member 可以不依赖任何项目，直接通过 Sidebar「新建任务」区域创建独立任务。独立任务同样由 WorkAgent 驱动、Sub-Agent 执行，与项目内任务的执行机制完全一致，区别仅在于它不归属于任何项目。

适用场景：临时性分析、个人头脑风暴、不成体系的单次任务，无需先创建项目再进入再创建任务。

**路由：** `/task/new` 或 `/task/:taskId`

**Sidebar 交互**

Sidebar 中「新建任务」是一个独立的可折叠分组，位于顶部导航（首页/Skills/Tools）与「我的项目」之间：

```
∨ 新建任务             [+]
    Q4 财报分析思路梳理        ← 历史独立任务（可点击进入）
    供应商评估框架设计
    数据清洗方法论对比
─────────────────────
我的项目              [+]
▼ 供应链金融风控      ●
    ...
```

- 点击区域标题右侧 `[+]` → 直接进入一个空白的新任务页
- 历史独立任务以对话标题展示，点击可重新进入
- 分组默认展开，可折叠

**与项目任务的区别**

| 维度 | 独立任务（新建任务）| 项目内任务 |
|------|----------------|----------|
| 创建权限 | 所有 Member 均可自主创建 | 仅项目负责人可创建 |
| 归属 | 不属于任何项目 | 归属某个项目 |
| 驱动 Agent | WorkAgent（临时实例）| WorkAgent（项目实例）|
| 上下文记忆 | 仅本次任务内（无跨任务记忆）| 共享项目 WorkAgent 的历史记忆 |
| 产出写入图谱 | 不写入任何项目图谱 | 自动写入项目知识图谱 |
| 成员协作 | 无协作，仅创建者可访问 | 项目成员可访问产出 |
| 审批 | 触发审批时，发回创建者自己的 HubAgent | 发送给项目负责人的 HubAgent |

**执行机制**

独立任务页与项目内任务页的界面和执行逻辑完全相同（WorkAgent + Sub-Agent 动态编排），不需要独立开发执行逻辑，复用任务页组件即可。

差异仅在于：WorkAgent 为临时实例，无项目历史上下文，任务结束后上下文不持久化到项目层。

**记忆归属**
- 独立任务的执行过程归创建者私有
- 不写入任何 WorkAgent 项目记忆
- 对话历史和产出文件归创建者个人，可在任务页查看

**权限**
- 所有 Member 均可创建独立任务，无需是项目负责人
- Admin 暂不支持（Admin 无 WorkAgent 入口）
- 独立任务对其他 Member 不可见

**验收标准**
- [ ] Sidebar「新建任务」分组位置正确（顶部导航与「我的项目」之间）
- [ ] 点击 `[+]` 进入空白新任务页，WorkAgent 就绪
- [ ] 历史独立任务按最近使用时间倒序展示在分组下
- [ ] 独立任务页复用项目内任务页的执行流组件
- [ ] 独立任务的审批请求发回创建者自己的 HubAgent（而非其他人）
- [ ] 任务产出不写入任何项目的知识图谱
- [ ] 其他 Member 无法访问该独立任务

### F-006 · HubAgent — 审批汇集与处理

**优先级：** P0
**模块：** HubAgent

**功能描述**
来自所有项目 WorkAgent 上报的审批请求，统一汇集在 HubAgent 对话区以审批卡片形式展示。这是项目负责人处理 Sub-Agent 审批的唯一主要入口。

**触发条件**
Sub-Agent 在执行任务时触发了审批策略（如调用外部 API 费用超阈值），WorkAgent 暂停该 Sub-Agent，将审批请求推送至项目负责人的 HubAgent。

**审批卡片**
```
[H] HubAgent  ·  审批请求

    来自：供应链项目 · chen 的任务「供应商信用评估」
    操作：调用企查查 API · 查询 47 家供应商
    预估费用：$240
    WorkAgent 建议：批准（历史通过率 95%，费用合理）

    [查看详情]        [拒绝]  [批准]
```

**处理流转**
```
Sub-Agent 触发审批
  ↓ WorkAgent 暂停 Sub-Agent
  ↓ 推送审批请求至项目负责人 HubAgent
HubAgent 展示审批卡片
  ↓ 负责人处理（批准 / 拒绝）
HubAgent 回传结果至 WorkAgent
  ↓ Sub-Agent 恢复执行（批准）或终止该分支（拒绝）
```

**记忆归属**
- 审批处理记录写入项目负责人的 HubAgent 记忆（个人记忆层）
- OrgAgent 可见审批操作日志（无内容详情，仅操作记录）

**权限**
- 仅项目负责人的 HubAgent 收到审批请求
- 普通 Member 的 HubAgent 不收到审批卡片
- Admin 若是项目负责人，审批卡片在项目页内展示（见 I-010）

**验收标准**
- [ ] 审批卡片在 WorkAgent 上报后实时出现
- [ ] 批准后 Sub-Agent 自动恢复执行
- [ ] 拒绝时支持填写原因，随结果回传
- [ ] 处理后卡片状态更新，KPI 卡片数字同步减少
- [ ] 普通 Member 的 HubAgent 不显示审批卡片

### F-007 · HubAgent — 主动摘要推送

**优先级：** P2
**模块：** HubAgent

**功能描述**
每日首次登录时，HubAgent 自动在对话区生成昨日进展摘要 + 今日待处理事项，帮助 Member 快速进入工作状态。

**推送内容**
```
[H] 早上好，samhar。

    昨日进展：
    · 「财报分析 Q4」已完成，产出 2 个文件
    · 「供应商评估」仍执行中，Sub-Agent 处理第 3/5 步

    今日待处理：
    · 1 个审批待处理（供应链项目 · chen 的任务）
    · lisa 的任务「应收账款预警」将于今日截止

    [查看审批]  [查看任务]
```

**触发规则**
- 每日首次访问 `/dashboard` 触发（非每次刷新）
- 距上次推送超过 8 小时才重新生成

**记忆归属**
- 摘要内容基于 HubAgent 记忆生成，不产生新的记忆写入

**权限**
- 仅 Member 账号触发，Admin 不触发（Admin 首页为 OrgAgent）

**验收标准**
- [ ] 每日首次登录触发，非每次刷新
- [ ] 摘要内容准确反映昨日任务状态
- [ ] 快捷操作正确跳转

### F-009 · OrgAgent 对话区

**优先级：** P1
**模块：** OrgAgent（Admin 首页右列）

**功能描述**
Admin 首页右列的 OrgAgent 对话区，提供全组织维度的战略层问答与效能管理能力。

**上下文范围**
- 聚合自所有 HubAgent 定期上报的 Summary（脱敏）
- Admin 手动输入的公司战略数据
- 全组织 Token 消耗、任务成功率、ROI 等统计指标
- 不含任何 Member 的个人对话内容、任务执行细节

**对话能力**
- 战略层分析：「哪个部门 AI 投入产出比最高」
- 跨团队对比：「供应链部和数据平台的任务成功率差异原因」
- 趋势预测：基于历史数据预测下月消耗，提前预警
- 效能建议：识别低效团队，主动推送优化建议（见 F-010）

### F-018 · 任务页 — Sub-Agent 审批中断与恢复

**优先级：** P0
**模块：** 任务页

**功能描述**
Sub-Agent 触发审批策略时，执行流中插入审批中断卡片，任务暂停。项目负责人在 HubAgent 处理审批后，Sub-Agent 自动恢复或终止。

**审批中断卡片（任务执行者视角）**
```
[W] WorkAgent  ·  需要审批

┌─────────────────────────────────────────────────────┐
│  ⏸ Sub-Agent [风险评估] 已暂停                      │
│  需要调用企查查 API                                  │
│  查询：47 家供应商  ·  预估费用 $240                 │
│  已通知 samhar（项目负责人）                         │
│                         [催审批]  [跳过此步骤]       │
└─────────────────────────────────────────────────────┘
```

**「催审批」**
- 重新发送飞书通知给负责人
- 冷却时间 1 小时，冷却中按钮灰色并显示剩余时间
- 防止重复打扰

**「跳过此步骤」**
- Sub-Agent 跳过需要审批的操作，继续执行后续步骤
- 任务完成汇总标注「已跳过：企查查 API 调用」
- 权限：仅任务执行者可操作（非负责人）

**审批结果更新（实时）**
- 批准：卡片更新为 `✓ 已批准 · samhar · 14:52`，Sub-Agent 自动继续
- 拒绝：卡片更新为 `✗ 已拒绝 · samhar · 原因：预算超限`，Sub-Agent 终止该分支

**权限**
- 「催审批」：任务执行者
- 「跳过」：任务执行者
- 审批处理（批准/拒绝）：项目负责人（在 HubAgent 处理，不在此页面）

**验收标准**
- [ ] 审批中断卡片在正确时机插入执行流
- [ ] 催审批冷却限制生效
- [ ] 跳过后任务继续，完成汇总有标注

```
[W] 收到。执行方案：
    Step 1  Sub-Agent[数据拉取]   → 飞书云文档
    Step 2  Sub-Agent[信用评估]   → 信用评估 Skill + 企查查 MCP
    Step 3  Sub-Agent[报告生成]   → 报告生成 Skill

    预计消耗：约 12K tokens · 预估费用：$18
    （基于历史 5 次同类任务估算，实际可能有 ±50% 浮动）

    [确认发起]  [修改方案]
```

确认后 WorkAgent 在 Sidebar 项目树中创建任务条目，WorkAgent 发送「任务已开始，[进入任务页]」，由用户主动决定是否跳转。

**权限**
- 所有项目成员均可请求 WorkAgent 生成方案
- 但只有项目负责人才能最终确认发起（因为任务创建权在负责人）
- 普通成员操作时：WorkAgent 生成方案后提示「需要负责人确认发起」

**验收标准**
- [ ] WorkAgent 能理解自然语言并生成合理执行方案
- [ ] 方案中的 Skill 和 Tool 均为当前项目可用的
- [ ] 消耗预估标注估算依据
- [ ] 普通成员尝试确认发起时收到权限提示

### F-023 · HubAgent — Skill 创建引导

**优先级：** P1
**模块：** HubAgent / Skills 页

**功能描述**
Member 在 Skills 页通过 HubAgent 对话描述需求，HubAgent 生成 Skill 配置草稿，用户确认后写入 Skills 库。新建 Skill 默认为私有，可后续申请升级。

**创建流程**
```
用户：我需要一个能自动生成财务摘要报告的 Skill

[H] 请确认以下配置：
    名称：财务摘要生成
    输入：财务数据文件（CSV / Excel）
    输出：结构化摘要报告（Markdown）
    执行逻辑：提取关键指标 → 计算同比环比 → 生成摘要
    可见范围：[私有 ▼]

    [确认创建]  [修改配置]
```

**Skill 默认私有**
新建 Skill 默认可见范围为「私有」，仅创建者可用。如需扩大范围，走 F-024 分层升级流程。

**记忆归属**
- 私有 Skill 归创建者个人所有
- 升级为项目共享后归项目所有
- 升级为全局预设后由 AgentOS 产品团队管理

**权限**
- 所有 Member 均可创建 Skill
- 创建后 Skill 立即可被该用户的 Sub-Agent 调用

**验收标准**
- [ ] HubAgent 能准确理解需求并生成合理 Skill 配置
- [ ] 确认后 Skill 立即出现在列表，可被 Sub-Agent 调用
- [ ] 新建 Skill 默认可见范围为私有

### F-024 · Skill 分层管理

**优先级：** P2
**模块：** Skills

**功能描述**
用户自建 Skill 支持三级可见范围，可逐级升级：私有 → 项目内共享 → 全局预设。

**升级路径**

| 升级方向 | 申请人 | 审批人 | 说明 |
|---------|--------|--------|------|
| 私有 → 项目内共享 | Skill 创建者 | 项目负责人 | 负责人批准后，项目内所有成员可用 |
| 项目内共享 → 全局预设 | 创建者或负责人 | AgentOS 产品团队 | 经官方评审后升级，获得官方背书 |

**降级**
- 全局预设 → 项目共享：仅 AgentOS 产品团队可操作
- 项目共享 → 私有：创建者或负责人可操作

**权限**
- 普通 Member：申请升级到项目内共享
- 项目负责人：批准 / 拒绝项目内共享申请
- Admin：查看全组织 Skill 使用统计，不参与审批流程
- AgentOS 产品团队：在 Agent 管理平台审批全局预设申请

**验收标准**
- [ ] 创建者可申请升级到项目内共享
- [ ] 负责人收到申请通知，可批准或拒绝
- [ ] 升级后可见范围实时生效
- [ ] 申请全局预设后进入产品团队审批队列

### F-026 · HubAgent — Tool 连接配置与权限控制

**优先级：** P1
**模块：** HubAgent / Tools 页

**功能描述**
Member 通过 HubAgent 对话完成 Tool 的连接配置，设置 Sub-Agent 的调用权限（费用上限触发审批），查询历史调用费用。

**连接配置（对话引导）**
```
用户：我想连接企查查

[H] 企查查使用 MCP 协议，需要你的 API Key。
    请粘贴在这里：[______________]
    连接后可设置 Sub-Agent 单次调用金额上限。
    [完成配置]
```

**调用权限控制**
```
用户：企查查单次调用超过 $50 需要我审批

[H] 已设置企查查 MCP 调用策略：
    预估费用 ≤ $50 → Sub-Agent 自动执行
    预估费用 > $50 → 暂停，发送审批请求到你的 HubAgent
```

此规则写入该 Member 在所有项目中的 Sub-Agent 审批策略，跨项目生效。

**调用费用查询**
```
用户：上个月我的 Sub-Agent 花了多少钱

[H] 上月 Tool 调用费用：
    企查查 MCP    $186  (142 次)
    其他 MCP      $24   (38 次)
    合计          $210
```

**记忆归属**
- Tool 配置和调用记录存入 HubAgent 记忆（个人私有）
- 费用统计数据脱敏后可上报 OrgAgent（统计层面）

**权限**
- 个人 Tool 配置仅本人可修改
- 费用上限规则跨该 Member 的所有项目生效

**验收标准**
- [ ] HubAgent 引导完成四类 Tool 的连接配置
- [ ] 费用阈值设置后在 Sub-Agent 执行中生效
- [ ] 调用记录查询数据准确

### F-027 · Sub-Agent 失败重试与上报

**优先级：** P1
**模块：** 任务页

**功能描述**
Sub-Agent 执行失败时，WorkAgent 自动重试（最多 3 次）。超出上限后在执行流中插入失败卡片，提供操作选项。

**重试机制**
- 第 1-3 次失败：WorkAgent 自动重试，任务树节点显示重试进度
- 第 3 次仍失败：插入失败卡片，通知执行者和项目负责人

**失败卡片**
```
[W] WorkAgent  ·  执行失败

┌─────────────────────────────────────────────────────┐
│  ✗ Sub-Agent [信用评估] 已失败（重试 3 次）          │
│  原因：企查查 API 返回超时                           │
│                                                     │
│           [查看详情]  [重试]  [放弃此步骤]           │
└─────────────────────────────────────────────────────┘
```

**通知**
- 任务执行者收到站内消息
- 项目负责人收到站内消息

**权限**
- `[重试]` `[放弃此步骤]`：任务执行者操作

**验收标准**
- [ ] 失败后自动重试，任务树显示重试状态
- [ ] 3 次重试失败后插入失败卡片
- [ ] 执行者和负责人均收到通知
